## Функциональные требования: объект Шаблон уведомления

### Входные UC

- UC-6.1 «Создать шаблон уведомления»
- UC-6.2 «Просмотреть шаблон уведомлений»
- UC-6.3 «Изменить шаблон уведомления»
- UC-6.4 «Удалить шаблон уведомления»
- UC-6.5 «Просмотреть список шаблонов уведомле»

### Кандидаты ФТ (иерархия Роль → Объект.Операция)

#### Управляющий, Владелец

- ШаблонУведомления.Создать

  Система должна позволять пользователю с ролью Управляющий, Владелец создавать объект Шаблон уведомления со следующими атрибутами:

  - идентификатор
  - код
  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Просмотреть

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать объект Шаблон уведомления со следующими атрибутами:

  - идентификатор
  - код
  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Изменить

  Система должна позволять пользователю с ролью Управляющий, Владелец изменять объект Шаблон уведомления со следующими атрибутами:

  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Удалить

  Система должна позволять пользователю с ролью Управляющий, Владелец помечать объект Шаблон уведомления как удалённый либо недоступный для использования в новых уведомлениях с изменением следующих атрибутов:

  - признакАктуальности = false

- ШаблонУведомления.ПросмотретьСписок

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать список объектов Шаблон уведомления с отображением для каждого шаблона следующих атрибутов:

  - идентификатор
  - код
  - наименование
  - тип.

#### Система

- ШаблонУведомления.ПроверитьПлейсхолдеры.СД. тело

  Система должна изменять [ШаблонУведомления]:тело по следующему алгоритму:

  - при сохранении шаблона система должна анализировать тело и тема на наличие плейсхолдеров в согласованном формате (например, {{имяКлиента}}, {{номерБронирования}})
  - если в тело или тема обнаружены плейсхолдеры, отсутствующие в перечне допустимых для выбранного типа шаблона, система должна отклонять сохранение шаблона с указанием списка некорректных плейсхолдеров и не изменять сохранённое значение тело
  - если все плейсхолдеры корректны, система должна разрешать сохранение шаблона без дополнительных изменений значения тело.

- ШаблонУведомления.Создать.СД. идентификатор

  Система должна изменять [ШаблонУведомления]:идентификатор по следующему алгоритму:

  - [ШаблонУведомления]:идентификатор = max([ШаблонУведомления]:идентификатор) + 1.

### Проверка полноты набора ФТ

- Основные потоки UC-6.1–UC-6.5 покрыты:
  - создание шаблона уведомления
  - просмотр шаблона уведомления
  - изменение шаблона уведомления
  - удаление/отключение шаблона уведомления
  - просмотр списка шаблонов уведомлений.
- Потенциально отсутствующие ФТ (рекомендации к расширению):
  - ФТ по версионированию шаблонов и невозможности изменения шаблонов, уже использованных для отправленных уведомлений
  - ФТ по фильтрации и поиску шаблонов в списке по типу, коду и статусу актуальности.
