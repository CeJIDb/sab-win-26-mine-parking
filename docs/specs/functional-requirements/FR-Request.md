## Функциональные требования: объект Обращение

### Входные UC

- UC-4.1 «Создать обращение»
- UC-4.2 «Просмотреть обращение»
- UC-4.3 «Просмотреть список обращений»
- UC-4.4 «Просмотреть обращение клиента»
- UC-4.5 «Ответить на обращение клиента»
- UC-4.6 «Просмотреть список обращений клиентов»

### Кандидаты ФТ (иерархия Роль → Объект.Операция)

#### КлиентФЛ, КлиентЮЛ

- Обращение.Создать

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ создавать объект Обращение со следующими атрибутами:

  - идентификатор
  - клиентИд
  - тип
  - канал
  - тема
  - описание
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

- Обращение.Просмотреть

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать принадлежащий ему объект Обращение со следующими атрибутами:

  - идентификатор
  - клиентИд
  - сотрудникИд
  - тип
  - канал
  - тема
  - описание
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

- Обращение.ПросмотретьСписок

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать список принадлежащих ему объектов Обращение с отображением для каждого обращения следующих атрибутов:

  - идентификатор
  - тип
  - тема
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

#### Управляющий, Владелец

- Обращение.Просмотреть

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать объект Обращение со следующими атрибутами:

  - идентификатор
  - клиентИд
  - сотрудникИд
  - тип
  - канал
  - тема
  - описание
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

- Обращение.Ответить

  Система должна позволять пользователю с ролью Управляющий, Владелец изменять объект Обращение со следующими атрибутами:

  - сотрудникИд
  - статус

- Обращение.ПросмотретьСписок

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать список объектов Обращение с отображением для каждого обращения следующих атрибутов:

  - идентификатор
  - клиентИд
  - сотрудникИд
  - тип
  - канал
  - тема
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

#### Система

- Обращение.ОбновитьСтатус.СД

  Система должна изменить Обращение со следующими атрибутами:

  - статус

- Обращение.ОбновитьСтатус.СД. статус

  Система должна изменять [Обращение]:статус по следующему алгоритму:

  - если по обращению создан ответ сотрудника и [Обращение]:статус ∈ {«новое», «в работе»}, то [Обращение]:статус = «ожидает клиента»
  - если клиент подтвердил решение или явно закрыл обращение, то [Обращение]:статус = «закрыто»
  - если с момента перевода статуса в «ожидает клиента» прошло более 14 календарных дней без активности клиента, то [Обращение]:статус = «решено».

- Обращение.Создать.СД. идентификатор

  Система должна изменять [Обращение]:идентификатор по следующему алгоритму:

  - [Обращение]:идентификатор = max([Обращение]:идентификатор) + 1.

### Проверка полноты набора ФТ

- Основные потоки UC-4.1–UC-4.6 покрыты:
  - создание обращения клиентом
  - просмотр обращения клиентом
  - просмотр списка обращений клиентом
  - просмотр обращения клиентом в админке управляющим/владельцем
  - ответ на обращение и изменение статуса
  - просмотр списка обращений в админке.
- Потенциально отсутствующие ФТ (рекомендации к расширению):
  - ФТ по фильтрации и поиску обращений в списках по типу, статусу, связанному объекту
  - ФТ по добавлению и просмотру истории изменений/комментариев в рамках одного обращения.
