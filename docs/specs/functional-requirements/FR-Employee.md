## Функциональные требования: объект Сотрудник

### Входные UC

- UC-15.1 «Создать профиль»
- UC-15.2 «Просмотреть профиль»
- UC-15.3 «Изменить профиль»
- UC-15.4 «Просмотреть список профилей»
- UC-15.5 «Авторизоваться»

### Кандидаты ФТ (иерархия Роль → Объект.Операция)

#### Управляющий, Владелец

- Сотрудник.Создать

  Система должна позволять пользователю с ролью Управляющий, Владелец создавать объект Сотрудник со следующими атрибутами:

  - идентификатор
  - логин
  - парольХэш
  - роль
  - фамилия
  - имя
  - отчество
  - телефон
  - email
  - статус

- Сотрудник.Просмотреть

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать объект Сотрудник со следующими атрибутами:

  - идентификатор
  - логин
  - роль
  - фамилия
  - имя
  - отчество
  - телефон
  - email
  - статус

- Сотрудник.Изменить

  Система должна позволять пользователю с ролью Управляющий, Владелец изменять объект Сотрудник со следующими атрибутами:

  - логин
  - роль
  - фамилия
  - имя
  - отчество
  - телефон
  - email

- Сотрудник.ИзменитьСтатус

  Система должна позволять пользователю с ролью Управляющий, Владелец изменять объект Сотрудник со следующими атрибутами:

  - статус

- Сотрудник.ПросмотретьСписок

  Система должна позволять пользователю с ролью Управляющий, Владелец просматривать список объектов Сотрудник с отображением для каждого сотрудника следующих атрибутов:

  - идентификатор
  - логин
  - роль
  - фамилия
  - имя
  - статус

#### Управляющий, Владелец, Охранник

- Сотрудник.Авторизоваться

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник авторизоваться по объекту Сотрудник со следующими атрибутами:

  - логин
  - пароль (для последующей проверки соответствия атрибуту парольХэш).

- Сотрудник.ИзменитьПароль

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник изменять свой объект Сотрудник со следующими атрибутами:

  - текущий пароль (для валидации)
  - новый пароль (для формирования нового значения атрибута парольХэш).

#### Система

- Сотрудник.ОбновитьСтатус.СД

  Система должна изменить Сотрудник со следующими атрибутами:

  - статус

- Сотрудник.ОбновитьСтатус.СД. статус

  Система должна изменять [Сотрудник]:статус по следующему алгоритму:

  - если количество неуспешных попыток авторизации по данному Сотрудник за последние 10 минут ≥ 5, то [Сотрудник]:статус = «заблокирован»
  - если выполнена успешная авторизация по данному Сотрудник и [Сотрудник]:статус = «заблокирован» по причине превышения числа попыток, то [Сотрудник]:статус = «активен».

- Сотрудник.Создать.СД. идентификатор

  Система должна изменять [Сотрудник]:идентификатор по следующему алгоритму:

  - [Сотрудник]:идентификатор = max([Сотрудник]:идентификатор) + 1.

### Проверка полноты набора ФТ

- Основные потоки UC-15.1–UC-15.5 покрыты:
  - создание профиля сотрудника в админке
  - просмотр профиля
  - изменение профиля
  - просмотр списка профилей
  - авторизация сотрудника.
- Потенциально отсутствующие ФТ (рекомендации к расширению):
  - дополнительные ФТ для явной блокировки/разблокировки сотрудника по решениям службы безопасности (отдельно от уже описанной автоматической блокировки)
  - ФТ по управлению распределением ролей и ограничениями доступа в разрезе связей Сотрудник ↔ Обращение, Уведомление, Парковочная сессия.
