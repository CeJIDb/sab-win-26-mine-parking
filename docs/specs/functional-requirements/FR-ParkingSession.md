- UC-12.4 «Создать ПС»
- UC-12.5 «Просмотреть активную ПС»
- UC-12.8 «Завершить ПС»
- UC-12.10 «Просмотреть список ПС»
- UC-12.11 «Просмотреть список ПС в админке»
- UC-12.12 «Просмотреть журнал въезда-выезда»
- UC-12.13 «Внести ТС в ПС вручную»

### Кандидаты ФТ (иерархия Роль → Объект.Операция)

#### КлиентФЛ, КлиентЮЛ

- ПарковочнаяСессия.ПросмотретьАктивную

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать активную для его ТС объект Парковочная сессия со следующими атрибутами:

  - идентификатор
  - бронированиеИд
  - времяВъезда
  - кппВъездаИд
  - способДопуска
  - статус

- ПарковочнаяСессия.ПросмотретьСписок

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать список объектов Парковочная сессия, связанных с его ТС/Бронированиями, с отображением для каждой ПС следующих атрибутов:

  - идентификатор
  - бронированиеИд
  - времяВъезда
  - времяВыезда
  - длительность
  - статус

#### Управляющий, Владелец, Охранник

- ПарковочнаяСессия.ПросмотретьСписокВАдминке

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник просматривать список объектов Парковочная сессия с отображением для каждой ПС следующих атрибутов:

  - идентификатор
  - бронированиеИд
  - времяВъезда
  - времяВыезда
  - длительность
  - кппВъездаИд
  - кппВыездаИд
  - способДопуска
  - сотрудникИд
  - статус

- ПарковочнаяСессия.ВнестиВручную

  Система должна позволять пользователю с ролью Охранник создавать объект Парковочная сессия вручную со следующими атрибутами:

  - идентификатор
  - бронированиеИд
  - времяВъезда
  - кппВъездаИд
  - способДопуска
  - сотрудникИд
  - статус

#### Система (СД-требования по Парковочная сессия)

Данные для автоматического создания и завершения ПС поступают из объекта СобытиеСКУД (см. FR-EventSKUD.md).

- ПарковочнаяСессия.Создать.СД

  Система должна создать объект Парковочная сессия, заполняя атрибуты по следующему алгоритму:

  - идентификатор — согласно ФТ ПарковочнаяСессия.Создать.СД. идентификатор
  - бронированиеИд = идентификатор активного Бронирование для данного ТС и парковки, определённого в UC-12.1/12.2
  - времяВъезда = [СобытиеСКУД]:датаВремяСобытия
  - кппВъездаИд = [СобытиеСКУД]:кппИд
  - способДопуска = «автоматически»
  - статус = «активна»

- ПарковочнаяСессия.Создать.СД. статус

  Система должна изменять [ПарковочнаяСессия]:статус по следующему алгоритму:

  - при создании ПС в UC-12.4 [ПарковочнаяСессия]:статус = «активна»

- ПарковочнаяСессия.Создать.СД. идентификатор

  Система должна изменять [ПарковочнаяСессия]:идентификатор по следующему алгоритму:

  - [ПарковочнаяСессия]:идентификатор = max([ПарковочнаяСессия]:идентификатор) + 1.

- ПарковочнаяСессия.Завершить.СД

  Система должна изменить Парковочная сессия по следующему алгоритму:

  - времяВыезда = [СобытиеСКУД]:датаВремяСобытия
  - длительность — согласно ФТ ПарковочнаяСессия.РассчитатьДлительность.СД. длительность
  - статус — согласно ФТ ПарковочнаяСессия.Завершить.СД. статус

- ПарковочнаяСессия.Завершить.СД. статус

  Система должна изменять [ПарковочнаяСессия]:статус по следующему алгоритму:

  - при фиксации выезда ТС в UC-12.8 [ПарковочнаяСессия]:статус = «завершена».

- ПарковочнаяСессия.РассчитатьДлительность.СД. длительность

  Система должна изменять [ПарковочнаяСессия]:длительность по следующему алгоритму:

  - если [ПарковочнаяСессия]:статус = «активна», то [ПарковочнаяСессия]:длительность = округлённая до минут разница между текущим системным временем и временемВъезда
  - если [ПарковочнаяСессия]:статус = «завершена», то [ПарковочнаяСессия]:длительность = округлённая до минут разница между временемВыезда и временемВъезда
  - если времяВыезда < времяВъезда (для завершённой ПС) или текущаяДатаВремя < времяВъезда (для активной ПС), система должна отклонять операцию расчёта длительности и фиксировать ошибку в журнале аудита, не изменяя длительность.

### Проверка полноты набора ФТ

- Основные потоки UC-12.4, UC-12.5, UC-12.8, UC-12.10–12.13 покрыты:
  - автоматическое создание и завершение ПС
  - просмотр активной ПС и списка ПС клиентом
  - просмотр списка ПС в админке и ручное создание ПС охранником.
- Потенциально отсутствующие ФТ (рекомендации к расширению):
  - СД‑ФТ по расчёту аналитической суммы по факту по ПС (если потребуется хранить её отдельно от суммы по бронированию)
  - ФТ по фильтрации и поиску ПС в списках по периоду, ТС, клиенту, сектору/ПМ и статусу
  - ФТ по отображению ПС в аналитических отчётах (UC-16.x) с учетом длительности и статуса.
