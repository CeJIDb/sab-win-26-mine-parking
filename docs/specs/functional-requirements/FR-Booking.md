### Кандидаты ФТ (иерархия Роль → Объект.Операция)

#### КлиентФЛ, КлиентЮЛ

- Бронирование.Создать

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ создавать объект Бронирование со следующими атрибутами:

  - идентификатор
  - тсИд
  - секторИд
  - парковочноеМестоИд
  - договорИд
  - датаНачала
  - датаОкончания
  - длительность
  - тип
  - статус
  - тарифИд
  - суммаКОплате

- Бронирование.Просмотреть

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать принадлежащий ему объект Бронирование со следующими атрибутами:

  - идентификатор
  - тсИд
  - секторИд
  - парковочноеМестоИд
  - договорИд
  - датаНачала
  - датаОкончания
  - длительность
  - тип
  - статус
  - тарифИд
  - суммаКОплате

- Бронирование.Изменить

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ изменять принадлежащий ему объект Бронирование со следующими атрибутами:

  - секторИд
  - парковочноеМестоИд
  - датаНачала
  - датаОкончания
  - длительность

- Бронирование.Отменить

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ изменять принадлежащий ему объект Бронирование со следующими атрибутами:

  - статус

- Бронирование.ПросмотретьСписок

  Система должна позволять пользователю с ролью КлиентФЛ, КлиентЮЛ просматривать список принадлежащих ему объектов Бронирования с отображением для каждого бронирования следующих атрибутов:

  - идентификатор
  - тсИд
  - секторИд
  - парковочноеМестоИд
  - датаНачала
  - датаОкончания
  - длительность
  - тип
  - статус
  - суммаКОплате

#### Управляющий, Владелец, Охранник

- Бронирование.СоздатьВАдминке

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник создавать объект Бронирование от имени клиента со следующими атрибутами:

  - тсИд
  - секторИд
  - парковочноеМестоИд
  - договорИд
  - датаНачала
  - датаОкончания
  - тарифИд

- Бронирование.СоздатьВАдминке.СД

  Система должна изменить Бронирование со следующими атрибутами:

  - идентификатор
  - длительность
  - тип
  - статус
  - суммаКОплате

- Бронирование.СоздатьВАдминке.СД. идентификатор

  Система должна изменять [Бронирование]:идентификатор по следующему алгоритму:

  - [Бронирование]:идентификатор = max([Бронирование]:идентификатор) + 1.

- Бронирование.СоздатьВАдминке.СД. длительность

  Система должна изменять [Бронирование]:длительность по следующему алгоритму:

  - [Бронирование]:длительность = [Бронирование]:датаОкончания - [Бронирование]:датаНачала
  - пересчитать в минуты, часы, дни, месяцы в зависимости от получившейся длительности.

- Бронирование.СоздатьВАдминке.СД. тип

  Система должна изменять [Бронирование]:тип по следующему алгоритму:

  - если [Бронирование]:длительность < 24 ч, то [Бронирование]:тип = «краткосрочное»
  - если [Бронирование]:длительность ≥ 24 ч и существует Договор, связанный с [Бронирование], то [Бронирование]:тип = «долгосрочное»

- Бронирование.СоздатьВАдминке.СД. статус

  Система должна изменять [Бронирование]:статус по следующему алгоритму:

  - [Бронирование]:статус = «ожидает оплаты».

- Бронирование.СоздатьВАдминке.СД. суммаКОплате

  Система должна изменять [Бронирование]:суммаКОплате по следующему алгоритму:

  - [Бронирование]:суммаКОплате рассчитывается по [Бронирование]:тарифИд и [Бронирование]:длительность
  - результат округляется до минимальной денежной единицы (например, до копеек).

- Бронирование.ПросмотретьВАдминке

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник просматривать объект Бронирование со следующими атрибутами:

  - идентификатор
  - тсИд
  - секторИд
  - парковочноеМестоИд
  - договорИд
  - датаНачала
  - датаОкончания
  - длительность
  - тип
  - статус
  - тарифИд
  - суммаКОплате

- Бронирование.ИзменитьВАдминке

  Система должна позволять пользователю с ролью Управляющий, Владелец, Охранник изменять объект Бронирование со следующими атрибутами:

  - секторИд

  - датаНачала
  - датаОкончания

- Бронирование.ИзменитьВАдминке.СД

  Система должна изменять Бронирование со следующими атрибутами:

  - длительность
  - статус

- Бронирование.ИзменитьВАдминке.СД. длительность

  Система должна изменять [Бронирование]:длительность по следующему алгоритму:

  - [Бронирование]:длительность = [Бронирование]:датаОкончания - [Бронирование]:датаНачала
  - пересчитать в минуты, часы, дни, месяцы в зависимости от получившейся длительности

- Бронирование.ИзменитьВАдминке.СД. статус

  Система должна изменять [Бронирование]:статус по следующему алгоритму:

  - если существует Платеж, связанный с [Бронирование], и [Платеж]:статус = «успешен», то [Бронирование]:статус = «подтверждено»
  - если текущее системное время ≥ [Бронирование]:датаНачала и [Бронирование]:статус = «подтверждено» и существует ПарковочнаяСессия, связанная с [Бронирование], то [Бронирование]:статус = «активно»
  - если текущее системное время ≥ [Бронирование]:датаОкончания и [Бронирование]:статус = «активно», то [Бронирование]:статус = «завершено»
  - если текущее системное время ≥ [Бронирование]:датаОкончания и [Бронирование]:статус = «подтверждено», то [Бронирование]:статус = «просрочено»

- Бронирование.ОтменитьВАдминке

#### Система (СД-требования по Бронирование)

Данные для создания авто-бронирования при въезде поступают из объекта СобытиеСКУД (см. FR-EventSKUD.md).

- Бронирование.СоздатьАвтоВъезд.СД

  Система должна создать объект Бронирование при автоматическом въезде ТС на основании принятого СобытиеСКУД со следующими атрибутами:

  - идентификатор
  - тсИд
  - секторИд
  - парковочноеМестоИд
  - датаНачала
  - тип
  - статус
  - тарифИд

- Бронирование.СоздатьАвтоВъезд.СД. идентификатор

  Система должна изменять [Бронирование]:идентификатор по следующему алгоритму:

  - [Бронирование]:идентификатор = max([Бронирование]:идентификатор) + 1

- Бронирование.СоздатьАвтоВъезд.СД. тсИд

  Система должна изменять [Бронирование]:тсИд по следующему алгоритму:

  - [Бронирование]:тсИд = идентификатор ТС, найденного по [СобытиеСКУД]:ГРЗ в UC-12.1

- Бронирование.СоздатьАвтоВъезд.СД. секторИд

  Система должна изменять [Бронирование]:секторИд по следующему алгоритму:

  - система определяет по выбранному Тарифу и типу ТС множество допустимых Типов зон
  - система формирует множество Секторов, у которых Тип зоны входит в множество допустимых Типов зон и в которых есть хотя бы одно свободное ПМ
  - если для типа клиента или ТС определены приоритетные зоны, система ограничивает множество Секторов только приоритетными Секторами (если такие есть)
  - из оставшихся Секторов система выбирает Сектор, ближайший к КПП, по которому получено СобытиеСКУД (по [СобытиеСКУД]:кппИд); при равном расстоянии выбирается Сектор с наименьшей загрузкой (минимальной долей занятых ПМ)
  - [Бронирование]:секторИд = идентификатор выбранного Сектора

- Бронирование.СоздатьАвтоВъезд.СД. парковочноеМестоИд

  Система должна изменять [Бронирование]:парковочноеМестоИд по следующему алгоритму:

  - система формирует множество ПМ в Секторе с идентификатором [Бронирование]:секторИд, удовлетворяющих условиям:
    - ПМ свободно
  - из множества подходящих ПМ система выбирает одно ПМ по следующей стратегии:
    - выбирается ПМ с минимальным идентификатором внутри Сектора (наименьшее значение идентификатора ПМ среди подходящих ПМ)
  - [Бронирование]:парковочноеМестоИд = идентификатор выбранного ПМ

- Бронирование.СоздатьАвтоВъезд.СД. датаНачала

  Система должна изменять [Бронирование]:датаНачала по следующему алгоритму:

  - [Бронирование]:датаНачала = [СобытиеСКУД]:датаВремяСобытия

- Бронирование.СоздатьАвтоВъезд.СД. тип

  Система должна изменять [Бронирование]:тип по следующему алгоритму:

  - если Бронирование создаётся в UC-12.2 по результатам UC-12.1, то [Бронирование]:тип = «автоматическое»

- Бронирование.СоздатьАвтоВъезд.СД. статус

  Система должна изменять [Бронирование]:статус по следующему алгоритму:

  - при успешном создании Бронирования в момент въезда ТС [Бронирование]:статус = «активно»

- Бронирование.СоздатьАвтоВъезд.СД. тарифИд

  Система должна изменять [Бронирование]:тарифИд по следующему алгоритму:

  - система определяет тип ТС по объекту ТС, связанному с [Бронирование]:тсИд
  - система формирует множество краткосрочных Тарифов, доступных для данного типа ТС
  - для каждого Тарифа из множества система определяет, какие Типы зон ему соответствуют, и какие Секторы парковки относятся к этим Типам зон
  - из множества краткосрочных Тарифов система исключает Тарифы, для которых на текущей парковке нет ни одного Сектора с подходящим Типом зоны и хотя бы одним свободным ПМ
  - если после исключения осталось несколько Тарифов, система выбирает среди них Тариф с минимальной базовой стоимостью (по настройкам тарифа); при равной базовой стоимости выбирается Тариф с наименьшей продолжительностью минимального расчетного интервала
  - [Бронирование]:тарифИд = идентификатор выбранного краткосрочного Тарифа

- Бронирование.ЗавершитьАвтоВыезд.СД

  Система должна изменить Бронирование при автоматическом завершении на выезде со следующими атрибутами:

  - датаОкончания
  - длительность
  - статус

- Бронирование.ЗавершитьАвтоВыезд.СД. статус

  Система должна изменять [Бронирование]:статус по следующему алгоритму:

  - если тип Бронирования ∈ {«краткосрочное», «автоматическое»} и существует ПарковочнаяСессия, связанная с [Бронирование], такая, что [ПарковочнаяСессия]:статус = «завершена», то [Бронирование]:статус = «завершено»

- Бронирование.ЗавершитьАвтоВыезд.СД. датаОкончания

  Система должна изменять [Бронирование]:датаОкончания по следующему алгоритму:

  - [Бронирование]:датаОкончания = [СобытиеСКУД]:датаВремяСобытия

- Бронирование.ЗавершитьАвтоВыезд.СД. длительность

  Система должна изменять [Бронирование]:длительность по следующему алгоритму:

  - [Бронирование]:длительность = округлённая до минут разница между [Бронирование]:датаОкончания и [Бронирование]:датаНачала
  - если [Бронирование]:датаОкончания < [Бронирование]:датаНачала, система не должна изменять [Бронирование]:длительность и должна зафиксировать ошибку в журнале аудита

- Бронирование.РассчитатьСуммуКОплате.СД. суммаКОплате

  Система должна изменять [Бронирование]:суммаКОплате по следующему алгоритму:

  - базовая сумма рассчитывается по Тарифу, типу Бронирования, датеНачала и длительности
  - при наличии льготного тарифа или договора с особыми условиями применяется соответствующая скидка/надбавка
  - результат округляется до минимальной денежной единицы (например, до копеек) по правилам тарификации и сохраняется в [Бронирование]:суммаКОплате

- Бронирование.Создать.СД. идентификатор

  Система должна изменять [Бронирование]:идентификатор по следующему алгоритму:

  - [Бронирование]:идентификатор = max([Бронирование]:идентификатор) + 1

- Бронирование.ИзменитьАвтоВъезд.СД

  Система должна изменять Бронирование со следующими атрибутами:

  - длительность
  - суммаКОплате

- Бронирование.ИзменитьАвтоВъезд.СД. длительность

  Система должна изменять [Бронирование]:длительность по следующему алгоритму:

  - если [Бронирование]:статус ∈ {«подтверждено», «активно»}, то [Бронирование]:длительность = округлённая до минут разница между текущим системным временем и [Бронирование]:датаНачала
  - если текущаяДатаВремя < [Бронирование]:датаНачала, система не должна изменять [Бронирование]:длительность и должна зафиксировать ошибку в журнале аудита

- Бронирование.ИзменитьАвтоВъезд.СД. суммаКОплате

  Система должна изменять [Бронирование]:суммаКОплате по следующему алгоритму:

  - базовая сумма рассчитывается по краткосрочному Тарифу с идентификатором [Бронирование]:тарифИд и текущему значению [Бронирование]:длительность (пересчитанному в минимальные тарифные единицы в соответствии с настройками тарифа)
  - при наличии льготного тарифа или договора с особыми условиями для клиента по данному Бронированию к базовой сумме применяется соответствующая скидка или надбавка
  - результат округляется до минимальной денежной единицы (например, до копеек) по правилам тарификации и сохраняется в [Бронирование]:суммаКОплате

### Проверка полноты набора ФТ

- Основные потоки UC-7.x, UC-8.x, UC-10.x и UC-12.2/12.9 покрыты:
  - создание, просмотр, изменение, отмена и списки бронирований для клиента и админки
  - автоматическое создание и завершение бронирований при въезде/выезде
  - расчёт суммы к оплате по бронированию
  - Потенциально отсутствующие ФТ (рекомендации к расширению):
  - СД‑ФТ по дополнительным состояниям статуса (например, «ожидает оплаты», «просрочено») в связке с UC-10.x и уведомлениями
  - ФТ по фильтрации и поиску бронирований в списках по типу, статусу, периоду и связанным сущностям (ТС, договор, клиент)
  - ФТ по массовым операциям с бронированиями в админке (изменение статуса группы бронирований при технических работах и т.п.).
