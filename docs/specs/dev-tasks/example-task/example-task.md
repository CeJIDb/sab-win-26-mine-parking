# Пример постановки задачи на разработку

Источник: изображение `постановка задачи на разработку пример.jpg`.

## Требование

Отобразить список свободных мест на парковке.

## Декомпозиция по компонентам архитектуры

### База данных

Создать таблицы:

- `Parking_Spaces`
- `Status_spaces`
- `Sector`

с атрибутами и связями согласно прилагаемой физической схеме БД.

### Фронтенд

1. Создать макеты для формы отображения свободных мест на парковке на экране бронирования на основе прототипов и согласовать с аналитиком.
2. Сверстать форму отображения свободных мест по макетам.
3. При поиске свободных мест отправить запрос на бэкенд на получение количества свободных мест для выбранных критериев поиска.
4. При нажатии на кнопку `"Показать"` обеспечить интеграцию с бэкендом для получения данных по свободным местам.

### Бэкенд

1. Реализовать функционал определения количества свободных мест по следующей логике:
   - дата или период, заданные пользователем;
   - выбор способа отображения: сектор или количество;
   - если ранее использовался конкретный способ отображения, выбрать его;
   - количество мест.
2. При отображении формы передать на фронтенд данные для отображения:
   - идентификатор парковки, на которой сейчас находится пользователь;
   - период, для которого отображается список;
   - все места, доступные для бронирования.
3. При поступлении запроса на определение количества свободных мест:
   - проверить доступ учетной записи к данным;
   - отправить данные на фронтенд, включая все места, доступные для бронирования.
