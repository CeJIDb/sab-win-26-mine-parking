# Карта навигации

Навигационная карта (sitemap) структуры сайта и личных кабинетов. Текстовое представление по результатам чтения тайлов `карта_навигации_tiles/tile_r*_c*.png` (исходник: `Карта навигации.jpg`).

---

## Условные обозначения

| Цвет узла | Назначение |
|-----------|------------|
| Фиолетовый | Точка входа / главная (сайт‑визитка) |
| Зелёный | Публичные разделы и личный кабинет клиента |
| Оранжевый | Личный кабинет администратора и рабочие интерфейсы персонала |
| Розовый | Внешняя система (интеграция / аналитика) |

---

## 1. Сайт‑визитка (точка входа)

*Фиолетовый узел — публичный сайт и стартовая точка навигации.*

**Главная страница сайта‑визитки:**

- Краткое описание парковки и преимуществ сервиса
- Базовые маркетинговые блоки (как работает, для кого, преимущества)

**Переходы с главной сайта:**

- **Контакты**
  - Адрес, схема проезда
  - Телефон, email
- **Тарифы и условия**
  - Описание тарифных планов
  - Основные правила пользования парковкой
- **Количество свободных парковочных мест**
  - Карта по секторам
  - Индикация доступности мест в реальном времени (обзорный режим)
- **Регистрация нового клиента**
  - Форма регистрации (ФИО, контакты, пароль и т.п.)
  - Подтверждение регистрации (сообщение об успешной регистрации)
- **Авторизация клиента**
  - Вход по логину и паролю
  - Вход через SSO (Яндекс ID / VK ID / MAX)
  - Восстановление доступа (ссылка/процесс восстановления пароля)
- **Авторизация администратора / персонала**
  - Ссылка для входа в ЛК администратора
  - Ссылка для входа в рабочий интерфейс охранника КПП (если доступно из публичной части)

## 2. Личный кабинет клиента

*Зелёная ветка — клиентская зона после успешной авторизации.*

### 2.1 Главная страница ЛК клиента

**Содержимое главной страницы:**

- Текущий статус парковки (на парковке / не на парковке)
- Текущий договор / активный тариф / активная парковочная сессия
- Статус оплаты (нет долга / есть долг / ближайший платёж)
- Количество свободных парковочных мест (карта по секторам) — вид в разрезе, релевантный клиенту
- Краткие уведомления (о скором окончании сессии, истечении договора и др.)

### 2.2 Разделы ЛК клиента

- **Управление бронированием**
  - Создание нового бронирования
  - Просмотр и изменение активных бронирований
  - Отмена бронирований
- **Управление договорами**
  - Список действующих и архивных договоров
  - Просмотр деталей договора
  - Загрузка/просмотр файлов договора (если предусмотрено)
- **История парковочных сессий**
  - Список прошлых сессий
  - Фильтры по дате, ТС, статусу
  - Детали сессии (время въезда/выезда, стоимость, сектор и место)
- **Управление транспортными средствами**
  - Список добавленных ТС (гос. номер, марка/модель и т.п.)
  - Добавление нового ТС
  - Редактирование/удаление ТС
- **Настройки профиля**
  - Персональные данные клиента
  - Контактные данные и уведомления
  - Настройки безопасности (смена пароля, управление SSO‑привязками)
- **Платежи и счета**
  - История платежей
  - Счета к оплате
  - Ссылки на оплату / статусы транзакций
- **Обращения**
  - Создание обращений и работа с обратной связью (вопросы, претензии, инциденты)
  - Связь обращений с бронированиями, сессиями, платежами и чеками
- **Уведомления**
  - Лента уведомлений по событиям (договоры, бронирования, сессии, платежи, обращения)
  - Переходы из уведомлений в соответствующие разделы ЛК

## 3. Личный кабинет администратора

*Оранжевая ветка — административный интерфейс управления парковкой.*

### 3.1 Вход администратора

- **Страница авторизации ЛК администратора**
  - Вход по логину и паролю
  - Возможные дополнительные факторы (если предусмотрены)
- После успешного входа — переход на главную страницу админки.

### 3.2 Главная страница ЛК администратора

- **Главный дашборд**
  - Ключевые операционные показатели:
    - Общая заполняемость парковки
    - Нагрузка по секторам
    - Количество активных сессий
    - Краткая сводка по инцидентам/блокировкам (если есть)
  - Быстрые ссылки в основные разделы управления

### 3.3 Операционное управление

- **Управление секторами и парковочными местами**
  - Конфигурация секторов (создание/изменение/деактивация)
  - Настройка конкретных парковочных мест (активно/заблокировано, тип места)
  - Просмотр состояния мест в реальном времени
- **Управление тарифами**
  - Список тарифов (активные и архивные)
  - Создание и редактирование тарифов
  - Настройка правил применения тарифов (по времени, типу клиента и т.п.)
- **Управление договорами**
  - Список договоров с клиентами
  - Просмотр деталей и статусов
  - Старт/продление/расторжение договоров

### 3.4 Управление профилями и доступами

- **Управление профилями клиентов**
  - Поиск и просмотр карточек клиентов
  - Изменение статусов клиентов (активен/заблокирован и т.п.)
  - Коррекция данных по обращениям (если требуется)
- **Управление профилями администраторов и персонала**
  - Список админов и сотрудников
  - Роли и права доступа
  - Активация/деактивация учётных записей
- **Обращения и инциденты**
  - Поиск и обработка обращений клиентов и инцидентов (раздел `admin/requests.html`)
  - Связь с профилями клиентов, договорами, сессиями и платежами

### 3.5 Аналитика и внешние системы

- **Страница аналитики (внешняя система)** — *розовый узел, интеграция*
  - Переход из админки в аналитическую систему (BI‑отчёты, дашборды)
  - Передача агрегированных данных о заполняемости, выручке, нагрузке по времени и т.п.
- Возможные дополнительные интеграции (учёт, биллинг и др.) — при наличии на схеме, также как розовые узлы.

## 4. Рабочий интерфейс охранника КПП

*Оранжевая ветка, отдельная точка входа, связанная с административным контуром, но логически выделенная как рабочий интерфейс персонала.*

### 4.1 Вход охранника

- **Страница авторизации рабочего интерфейса КПП**
  - Логин и пароль сотрудника
  - Дополнительный код подтверждения (если предусмотрен)
- После успешного входа — переход в специализированный интерфейс КПП.

### 4.2 Рабочий интерфейс КПП

- **Панель проверки доступа**
  - Проверка права въезда по гос. номеру ТС
  - Проверка статуса договора/бронирования/сессии
- **Операции на КПП**
  - Регистрация факта въезда/выезда
  - Оформление разовых визитов (офлайн‑клиенты)
  - Отметка инцидентов, ручные корректировки при необходимости
- **Справочная информация**
  - Просмотр базовой информации о клиенте/ТС
  - Быстрый переход к карточке клиента в админке (по ссылке, если права позволяют)

## 5. Схема связей (дерево)

Текстовое дерево текущей навигации по новой схеме:

```
[Сайт‑визитка (фиолетовый)]
  ├─ Главная сайта
  │   ├─ Контакты
  │   ├─ Тарифы и условия
  │   ├─ Количество свободных парковочных мест (карта по секторам)
  │   ├─ Регистрация нового клиента
  │   ├─ Авторизация клиента
  │   │   ├─ Вход по логину и паролю
  │   │   └─ Вход через SSO (Яндекс ID / VK ID / MAX)
  │   └─ Авторизация администратора / персонала
  │       ├─ Вход в ЛК администратора
  │       └─ Вход в рабочий интерфейс КПП
  └─ (при необходимости) Прямые ссылки на публичные разделы

[Личный кабинет клиента (зелёный)]
  ├─ Главная ЛК клиента
  │   ├─ Статус парковки
  │   ├─ Текущий договор / тариф / сессия
  │   ├─ Статус оплаты / долг
  │   └─ Карта свободных мест (клиентский вид)
  ├─ Управление бронированием
  ├─ Управление договорами
  ├─ История парковочных сессий
  ├─ Управление транспортными средствами
  ├─ Настройки профиля
  ├─ Платежи и счета
  ├─ Обращения
  └─ Уведомления

[Личный кабинет администратора (оранжевый)]
  ├─ Авторизация ЛК администратора
  └─ Главная админки
      ├─ Дашборд (операционные показатели)
      ├─ Управление секторами и парковочными местами
      ├─ Управление тарифами
      ├─ Управление договорами
      ├─ Управление профилями клиентов
      ├─ Управление профилями админов и персонала
      ├─ Обращения и инциденты
      └─ Аналитика (розовый, внешняя система)
          └─ BI‑отчёты и дополнительные дашборды

[Рабочий интерфейс охранника КПП (оранжевый)]
  ├─ Авторизация рабочего интерфейса КПП
  └─ Рабочая панель КПП
      ├─ Проверка права въезда (гос. номер, договор, бронирование)
      ├─ Регистрация въезда/выезда
      └─ Оформление разовых визитов и инцидентов
```

## Соответствие артефактам и протоколам

- **Карта навигации (sitemap)** — текстовое отражение графической схемы `Карта навигации.jpg` и тайлов `карта_навигации_tiles/tile_r*_c*.png`.
- **User Story Map / Impact Map** — обосновывают выделение отдельных точек входа (ЛК клиента, ЛК администратора, рабочий интерфейс КПП).
- **Артефакты аналитики и интеграций** — розовые узлы схемы соответствуют внешней аналитической системе и связанным интеграциям.
