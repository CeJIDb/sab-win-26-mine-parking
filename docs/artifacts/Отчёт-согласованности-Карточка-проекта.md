# Отчёт о согласованности: Карточка проекта

## 1. Резюме

**Карточка проекта в целом соответствует** артифактам и протоколам по целям, заинтересованным сторонам, ограничениям MVP, каналам доступа и бюджету. Терминология глоссария (тариф, бронирование, договор, ГРЗ, ТС, ПМ, КПП, СКУД, LPR, ЛК) согласована с Протоколом №4 и User Story Map.

**Имеются частичные расхождения и пробелы:**
- В карточке не отражены типы зон, параметры ТС (тип/категория) и льготные категории клиентов, хотя они зафиксированы в Контекстной диаграмме, схеме AS-IS и User Story Map.
- В блоке «Смежные системы» не указаны ОФД и сервис уведомлений; не упомянуты информационные табло и дисплеи как оборудование на границе с платформой.
- Не зафиксирован срок реализации (4–6 мес. по протоколам и Opportunity Canvas).
- Возможность обращений/обратной связи для клиентов в карточке явно не выделена (есть в Impact Map и протоколе №5).
- Роль «старший охранник» в карточке не выделена (в контекстных диаграммах — отдельно).

Рекомендуется внести предложенные ниже правки в «Карточка-проекта.md» для полного выравнивания с артифактами и протоколами.

## 4. Что согласовано (без правок)

- **Цель проекта:** 80% клиентов через онлайн в течение 6 месяцев после запуска — совпадает с Impact Map, протоколами №3, №5, Opportunity Canvas.
- **Бизнес-эффекты:** снижение очередей, простой ≤10%, рейтинг ≥4.5, пропускная способность (целевая 200 машин/час) — совпадает с Opportunity Canvas и протоколом №5.
- **MVP:** заключение договоров с ЮЛ вне MVP; доступ ЮЛ через систему и самообслуживание по спискам ТС — в MVP. Совпадает с протоколом №4, Контекстной диаграммой MVP, User Story Map.
- **Каналы доступа:** клиенты — мобильные (PWA); охрана — ПК на КПП; управляющий/владелец — из любого места. Совпадает с протоколом №4.
- **Нецифровые клиенты:** сохранение возможности для ~20%; обязательная минимальная идентификация (телефон + имя). Совпадает с протоколом №5 и контекстными диаграммами.
- **Бухгалтерия:** внутренние процессы не меняются. Совпадает с протоколами.
- **Бюджет:** ~10 млн руб. на всё (включая оборудование), до 15 млн при обосновании. Совпадает с протоколом №5 и Opportunity Canvas.
- **Определения** тарифа, бронирования, договора по смыслу совпадают с протоколом №4 и User Story Map.
- **Аналитика в MVP:** сбор данных без «спец-аналитики» — совпадает с протоколом №4.

## 5. Конкретные правки для «Карточка-проекта.md»

Ниже — формулировки для вставки в документ.

### 5.1. Глоссарий (дополнение после строки с ЛК)

Добавить строки в таблицу глоссария:

```markdown
| **ОВЗ** | Ограниченные возможности здоровья; влияют на право парковки в специальных зонах и на применение скидок. |
| **Типы зон** | Краткосрочная парковка, долгосрочная (ФЛ+ЮЛ), места для лиц с ОВЗ, муниципальная техника + мотоциклы + резерв (по схеме AS-IS); тарифы и правила могут различаться по типу зоны. |
| **Параметры ТС (тип/категория)** | Легковой, мотоцикл, муниципальная техника и т.д.; используются для тарификации и размещения по зонам. |
| **Льготные категории** | ОВЗ, пенсионеры, ветераны и др.; влияют на доступ в соответствующие зоны и применение скидок. |
```
