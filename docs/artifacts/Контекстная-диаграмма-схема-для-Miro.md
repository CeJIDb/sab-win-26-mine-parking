# Схема контекстной диаграммы для рисования в Miro (DFD Level 0)

Краткая структура по документу `Контекстная диаграмма.md` (полная поставка). Один центральный процесс, вокруг — внешние сущности; стрелки подписать по потокам ниже.

---

## 1. Расположение элементов (как расставить в Miro)

**В центре — один процесс (прямоугольник или скруглённый блок):**

| Элемент | Подпись на блоке |
|--------|-------------------|
| Центральная система | **Цифровая платформа парковок** (процесс 0) |

**Внешние сущности — вокруг центра.** Рекомендуемое размещение (слева — акторы-источники, справа — получатели; сверху/снизу — системы):

| № | Сущность | Тип | Куда ставить | Подпись на блоке |
|---|----------|-----|----------------|-------------------|
| 1 | ЮЛ | Актор | Слева от центра | **ЮЛ** (юридическое лицо) |
| 2 | ФЛ | Актор | Слева от центра | **ФЛ** (физическое лицо) |
| 3 | Оператор КПП | Актор | Справа от центра | **Оператор КПП** (охранник / старший охранник) |
| 4 | Администратор | Актор | Справа от центра | **Администратор** (управляющий / владелец) |
| 5 | Платежная система | Внешняя система | Снизу (или справа) | **Платежная система** (онлайн) |
| 6 | Платежный терминал | Внешняя система | Снизу | **Платежный терминал** (офлайн) |
| 7 | ОФД | Внешняя система | Снизу | **ОФД** (оператор фискальных данных) |
| 8 | Сервис уведомлений | Внешняя система | Снизу | **Сервис уведомлений** |
| 9 | SSO (IdP) | Внешняя система | Снизу | **SSO** (только для ФЛ: Яндекс ID, VK ID, Госуслуги и т.п.) |
| 10 | СКУД с LPR | Внешняя система | Снизу | **СКУД с LPR-камерами** |
| 11 | Геосервисы и Сайт-визитка | Внешняя система | Сверху | **Геосервисы и Сайт-визитка** |
| 12 | ЭДО | Внешняя система | Снизу | **ЭДО** (электронный документооборот) |
| 13 | Биллинг/1С | Внешняя система | Снизу | **Биллинг и учёт (1С и др.)** |
| 14 | Аналитика | Внешняя система | Снизу | **Аналитика** (внешняя система) |
| 15 | Информационное табло КПП | Оборудование | Сверху или справа | **Информационное табло КПП** |
| 16 | Дисплей стойки на въезде | Оборудование | Справа | **Дисплей стойки на въезде** |
| 17 | Дисплей стойки на выезде | Оборудование | Справа | **Дисплей стойки на выезде** |

## 3. Стрелки и подписи потоков (кратко)

**Правило:** стрелка **к центру** = «в систему», стрелка **от центра** = «из системы». На Miro можно подписать стрелку одним обобщённым названием или вынести список в комментарий/стикер.

| От кого → в систему | Краткая подпись стрелки (в систему) | От системы → кому | Краткая подпись стрелки (из системы) |
|---------------------|--------------------------------------|-------------------|--------------------------------------|
| **ЮЛ** → центр | Профиль, данные организации, ГРЗ и ТС, ПМ, договор, оплата, бронирование, согласия, обращения, скан документов, льготная категория | Центр → **ЮЛ** | Тарифы и условия, доступность мест, договор и счета, оплата, ПМ и инструкции, уведомления, история, ответы по обращениям |
| **ФЛ** → центр | Профиль, льготная категория, ГРЗ и ТС, данные/параметры договора, оплата, бронирование, согласия, минимальная идентификация (гость), обращения, скан документов | Центр → **ФЛ** | Тарифы и условия, доступность мест, договор и счета, оплата, ПМ и инструкции, уведомления, история, ответы по обращениям |
| **Оператор** → центр | ГРЗ (ручной ввод), решение по доступу, поиск профиля, парковочная сессия, обработка исключений, инициация оплаты через терминал | Центр → **Оператор** | Обращения клиентов, данные договора и доступа, категория ТС, льготная категория, инструкции, статус распознавания ГРЗ |
| **Администратор** → центр | Профили, договоры и правила, доступ и тарифы (в т.ч. по зонам/ТС/льготам), территория и зоны, обращения, запросы отчётности | Центр → **Администратор** | Аналитика и отчётность, дашборды и выгрузки, подтверждения, уведомления о просрочках |
| **Платежная система** → центр | Статус платежа, id платежа, квитанция, статус возврата | Центр → **Платежная система** | Сумма к оплате, id платежа, сумма к возврату |
| **Платежный терминал** → центр | Статус платежа, id платежа, квитанция | Центр → **Платежный терминал** | Сумма к оплате, id платежа |
| **ОФД** → центр | Фискальный чек, статус фискализации | Центр → **ОФД** | Чек к фискализации, id платежа |
| **Сервис уведомлений** → центр | Статус отправки | Центр → **Сервис уведомлений** | OTP, уведомление, канал, адресат |
| **SSO/IdP** → центр *(только для входа ФЛ в ЛК)* | Результат аутентификации, идентификатор пользователя, атрибуты (контакты и т.д.) | Центр → **SSO/IdP** | Запрос на аутентификацию ФЛ, параметры запроса (client_id, scope и т.п.) |
| **СКУД** → центр | ГРЗ, вероятность, дата/время, полоса, номер КПП, id кадра; при наличии — шлагбаум, статус устройства | Центр → **СКУД** | Право доступа (разрешен/запрещен); при необходимости — ГРЗ для логирования, запрос повторного распознавания |
| **Геосервисы / Сайт-визитка** | — (только приём) | Центр → **Геосервисы / Сайт** | Информация о работе, правилах, тарифах; количество свободных мест |
| **ЭДО** | Подписанные документы от контрагентов, статусы | Центр → **ЭДО** | Подписанные договоры, счета, акты, исходящие документы |
| **Биллинг/1С** | Справочники, реквизиты, данные для сверки | Центр → **Биллинг/1С** | Выгрузки по оплатам, договорам, сессиям, клиентам |
| **Аналитика** | Отчёты, дашборды, выгрузки | Центр → **Аналитика** | Агрегированные данные (загрузка, выручка, договоры, бронирования, клиенты) |
| **Табло КПП** | — | Центр → **Табло** | Свободные места, статус КПП, инструкции, предупреждения, аварийные сообщения |
| **Дисплей въезд** | — | Центр → **Дисплей въезд** | ГРЗ, право доступа, инструкции, тариф, назначенные ПМ/сектор |
| **Дисплей выезд** | — | Центр → **Дисплей выезд** | ГРЗ, право доступа, тариф, сумма к оплате, статус платежа, инструкции |

## 4. Чек-лист для Miro

- [ ] Один центральный блок: **Цифровая платформа парковок**
- [ ] 4 актора: ЮЛ, ФЛ, Оператор КПП, Администратор
- [ ] 10 внешних систем: Платежная система, Платежный терминал, ОФД, Сервис уведомлений, **SSO (IdP)**, СКУД с LPR, Геосервисы и Сайт-визитка, ЭДО, Биллинг/1С, Аналитика
- [ ] 3 элемента оборудования: Табло КПП, Дисплей въезд, Дисплей выезд
- [ ] Стрелки к центру подписаны (в систему)
- [ ] Стрелки от центра подписаны (из системы)
- [ ] Оборудование: только стрелки от центра к табло/дисплеям (обратного потока нет)
