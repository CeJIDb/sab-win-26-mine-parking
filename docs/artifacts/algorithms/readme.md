## Назначение

Раздел нужен для артефактов, которые не являются основной бизнес-моделью, но помогают формализовать логику решения.

Сюда можно складывать DRAKON и другие алгоритмические схемы, если они используются как учебный, поясняющий или подготовительный материал.

## Рекомендуемый состав

Примеры файлов, которые логично хранить в этом разделе:

- `drakon-access-check.md`
- `drakon-parking-session-lifecycle.md`
- `algorithm-manual-access-decision.md`
- `drakon-parking-payment.md`

Если схема существует как изображение, размещайте его рядом с `md`-описанием.

Текущий каноничный артефакт раздела:

- [DRAKON-схема оплаты парковки](drakon-parking-payment.md) — формализует логику оплаты в виде алгоритмической схемы.
- [Индекс изображений алгоритмических артефактов](assets/readme.md) — содержит связанные изображения и приложения.

## Заметки по использованию

- Не переносите сюда каноничные требования, архитектурные решения или use case.
- Если алгоритм становится частью архитектурного решения, сошлитесь на соответствующий ADR, DDD или C4-документ.
- Если материал нужен только для демонстрации, оставьте каноничный файл здесь, а презентационную выжимку положите в `docs/demo-days/`.
