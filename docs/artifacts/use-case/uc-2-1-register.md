## Детализация содержания

### UC-2.1 «Зарегистрироваться»

**Действующее лицо:** Клиент

**Предусловия:**

Клиент открыл портал

**Триггер:** Постусловие: Клиент переходит к регистрации личного кабинета

**Основной поток:**

1. Пользователь открывает форму регистрации
2. Система отображает параметры для регистрации:

- Номер телефона
- ФИО
- E-mail

3. Система предлагает заполнить форму регистрации вручную
4. Пользователь заполняет форму регистрации
5. Система валидирует заполненные данные (на полноту и корректность)
6. Система проверят наличие данного клиента в системе
7. Система создает нового клиента
8. Система отображает сообщение об успешном завершении процесса создания
9. Клиент просмотрел сообщение
10. Сценарий завершен

**Расширение:**

**3а. Пользователь отказался от заполнения формы вручную.**
3а1. Система запрашивает разрешение на получение данных через внешние сервисы:

- Госуслуги
- Вконтакт
- ЯндексID

3а2. Пользователь соглашается и выбирает один из предложенных сервисов
3а3. Система запрашивает параметры в указанном сервисе
3а4. Система получает успешный ответ от сервиса
3а5. Система подставляет полученные значения в параметры для регистрации
3а6. Сценарий переходит к шагу 5
