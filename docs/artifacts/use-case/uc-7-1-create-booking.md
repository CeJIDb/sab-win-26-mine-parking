## Детализация содержания

### UC-7.1 «Создать бронирование»

**Действующее лицо:** Клиент

**Предусловия:**

- Клиент зарегистрирован в системе и авторизован в личном кабинете.
- Система доступна.
- Раздел «Бронирования» активен.
- В системе настроены тарифы и правила бронирования (допустимые даты/диапазон времени, лимиты по длительности, шаг интервала).
- В профиле клиента есть хотя бы одно ТС, если бронь привязывается к конкретному автомобилю.
- Для выбранного тарифа предусмотрен хотя бы один доступный способ оплаты (если бронь требует оплаты для подтверждения).
- У клиента нет другой активной брони на тот же период для того же ТС

**Триггер:** Пользователь в личном кабинете инициирует создание новой брони (открывает форму бронирования).

**Постусловие:**

В системе создана бронь с выбранными параметрами;

- если оплата выполнена в рамках сценария, бронь имеет статус «подтверждена», а место/слот окончательно закреплены за клиентом на указанный период;
- если оплата отложена, бронь имеет статус «ожидает оплаты» и будет автоматически отменена по истечении времени резерва согласно правилам.

**Основной поток:**

1. Пользователь открывает форму бронирования.
2. Система отображает форму бронирования с полями:
   - дата и время начала
   - длительность
   - ТС
   - сектор
   - тариф
3. Пользователь заполняет форму бронирования.
4. Система валидирует заполненные данные.
5. Система проверяет наличие свободного места для бронирования.
6. Система отображает возможность бронирования.
7. Система отображает стоимость бронирования.
8. Пользователь подтверждает бронирование.
9. Система резервирует ПМ в соответствии с параметрами брони на N 15 минут.
10. Система отображает параметры и стоимость бронирования.
11. Пользователь оплачивает бронирование.
12. Система получает подтверждение оплаты.
13. Система изменяет статус слота в секторе на Подтверждено.
14. Система уведомляет пользователя об успешном бронировании.
15. Сценарий окончен.

**Расширение:**

4.а. Не заполнены обязательные поля  
4.а.1. Система обнаруживает, что одно или несколько обязательных полей формы пусты.  
4.а.2. Система подсвечивает незаполненные поля и выводит подсказку: «Заполните обязательные поля».  
4.а.3. Сценарий возвращается к шагу 3 Основного потока.

4.б. Некорректные значения полей (диапазон дат/времени)  
4.б.1. Система определяет, что дата начала в прошлом, дата окончания раньше даты начала или длительность выходит за допустимые лимиты.  
4.б.2. Система отображает сообщение об ошибке: «Укажите корректные дату и время бронирования».  
4.б.3. Сценарий возвращается к шагу 3 Основного потока.

5.а. Нет свободных мест на выбранный период  
5.а.1. Система не находит свободных мест (или квоты) на указанные дату, время и сектор.  
5.а.2. Система отображает сообщение: «На выбранный период нет свободных мест. Выберите другие дату/время или сектор».  
5.а.3. Сценарий возвращается к шагу 3 Основного потока.

9.а. Есть пересекающаяся бронь у клиента  
9.а.1. Система обнаруживает, что у клиента уже есть активная бронь на тот же период для выбранного ТС.  
9.а.2. Система отображает сообщение: «У вас уже есть бронь на это время для данного автомобиля».  
9.а.3. Сценарий завершается или возвращается к шагу 3 Основного потока для изменения параметров.

11.а. Ошибка оплаты
11.а.1. Платёжный шлюз возвращает ошибку.  
11.а.2. Система не выполняет окончательное закрепление места (шаг 13), оставляет бронь в статусе «ожидает оплаты» в пределах времени резерва и отображает сообщение: «Оплата не выполнена. Повторите попытку или бронь будет отменена».  
11.а.3. Клиент может повторить оплату до истечения времени резерва; при его истечении бронь автоматически отменяется по регламенту.

11.б. Отмена оплаты
11.б.1. Клиент отменяет оплату.  
11.б.2. Система не выполняет окончательное закрепление места (шаг 13), оставляет бронь в статусе «ожидает оплаты» в пределах времени резерва и отображает сообщение: «Оплата не выполнена. Повторите попытку или бронь будет отменена».  
11.б.3. Клиент может повторить оплату в течение 15 минут, по истечения времени бронирование аннулируется и сценарий заканчивается.
