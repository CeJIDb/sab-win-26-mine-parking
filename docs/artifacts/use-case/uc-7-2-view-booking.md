## Детализация содержания

### UC-7.2 «Просмотреть бронирование»

**Действующее лицо:** Клиент ФЛ, Клиент ЮЛ

**Предусловия:**

- Клиент зарегистрирован в системе и авторизован в личном кабинете.
- Система доступна.
- Раздел "Бронирования" активен.
- В системе существует хотя бы одно бронирование, привязанное к аккаунту клиента.

**Триггер:** Клиент выбирает конкретное бронирование из списка бронирований (UC-7.5) в личном кабинете.

**Постусловие:**

- Клиент видит детальную информацию о выбранном бронировании.
- Для бронирований со статусом "Ожидает оплаты" доступна кнопка "Оплатить".

**Основной поток:**

1. Пользователь выбирает бронирование из списка.
2. Система отображает детальную карточку бронирования:

- номер бронирования;
- тип бронирования (краткосрочное / долгосрочное / автоматическое);
- статус (Ожидает оплаты / Активно / Подтверждено / Завершено / Отменено);
- тариф;
- назначенный сектор / парковочное место;
- ТС, привязанное к бронированию;
- дата и время начала и окончания;
- стоимость бронирования;
- статус оплаты и фактически оплаченная сумма.

3. Для долгосрочных бронирований система дополнительно отображает:

- договор, на основании которого создано бронирование (номер, ссылка).

4. Для бронирований со статусом "Ожидает оплаты" система дополнительно отображает:

- время, оставшееся до автоматической отмены;
- кнопку "Оплатить".

5. Сценарий окончен.

**Расширение:**

**1а. Бронирование не найдено или не принадлежит аккаунту клиента.**
1а1. Система отображает сообщение "Бронирование не найдено".
1а2. Сценарий завершается.
