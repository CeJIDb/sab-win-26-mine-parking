## Детализация содержания

### UC-7.5 «Просмотреть список бронирований»

**Действующее лицо:** Клиент ФЛ, Клиент ЮЛ

**Предусловия:**

- Клиент зарегистрирован в системе и авторизован в личном кабинете.
- Система доступна.
- Раздел "Бронирования" активен.

**Триггер:** Клиент переходит в раздел "Мои бронирования" в личном кабинете.

**Постусловие:**

Клиент видит список своих бронирований, отфильтрованный и/или отсортированный по
выбранным параметрам.

**Основной поток:**

1. Система загружает и отображает список бронирований клиента.
2. Для каждой записи в списке система отображает:

- Идентификатор;
- ГРЗ;
- Сектор;
- Номер парковочного места;
- Дата начала;
- Дата окончания;
- Длительность;
- Тип бронирования;
- Статус;
- Сумма к оплате.

3. Список по умолчанию отсортирован по дате начала (новые — первыми).
4. Клиент может применить фильтр по статусу или временному периоду.
5. Клиент может выбрать конкретное бронирование — система выполняет UC-7.2.
6. Сценарий окончен.

**Расширение:**

**1а. У клиента нет ни одного бронирования.**
1а1. Система отображает сообщение "У вас пока нет бронирований".
1а2. Система отображает кнопку "Создать бронирование" (переход в UC-7.1).
1а3. Сценарий завершается.

**4а. После применения фильтра список оказался пустым.**
4а1. Система отображает сообщение "По заданным фильтрам бронирований
не найдено".
4а2. Сценарий возвращается к шагу 4 для изменения фильтра.
