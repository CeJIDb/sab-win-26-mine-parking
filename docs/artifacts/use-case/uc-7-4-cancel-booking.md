## Детализация содержания

### UC-7.4 «Отменить бронирование»

**Действующее лицо:** Клиент ФЛ, Клиент ЮЛ

**Предусловия:**

- Клиент зарегистрирован в системе и авторизован в личном кабинете.
- Система доступна.
- В системе существует бронирование клиента со статусом "Ожидает оплаты" или "Подтверждено".
- Парковочная сессия по данному бронированию ещё не создана.

**Триггер:** Клиент открывает карточку бронирования (UC-7.2) и нажимает кнопку "Отменить бронирование".

**Постусловие:**

Вариант "отмена выполнена":

- статус бронирования изменён на "Отменено";
- парковочное место / сектор освобождены в системе.
  Вариант "отмена не выполнена":
- бронирование остаётся в исходном состоянии.

**Основной поток:**

1. Система отображает диалог подтверждения отмены с предупреждением об условиях.
2. Клиент подтверждает отмену.
3. Система изменяет статус бронирования на "Отменено".
4. Система освобождает зарезервированное место / сектор.
5. Система отправляет клиенту уведомление об отмене бронирования.
6. Система отображает клиенту подтверждение: "Бронирование успешно отменено".
7. Сценарий окончен.

**Расширение:**

1.а. Бронирование не допускает отмены клиентом (парковочная сессия уже создана).
1.а.1. Система скрывает кнопку "Отменить бронирование".
1.а.2. Сценарий завершается.
2.а. Клиент отказывается от подтверждения отмены.
2.а.1. Система закрывает диалог подтверждения, бронирование остаётся без
изменений.
2.а.2. Сценарий завершается.
3.а. Оплата по бронированию уже получена (статус "Подтверждено").
3.а.1. После изменения статуса на "Отменено" система инициирует процесс
возврата средств (UC-10.5).
3.а.2. Сценарий продолжается с шага 4.
