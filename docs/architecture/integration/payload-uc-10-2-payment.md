Документ нужен, чтобы:

- дать разработчику и QA каноничные примеры для реализации и проверок;
- служить опорой для генерации мок-ответов в тестовых окружениях.

## Контекст применения

Ответ относится к процессу `UC-10.2-1 — Обработка оплаты парковочной сессии`. Он возвращается платежным сервисом после завершения операции оплаты у внешнего платежного провайдера.

Корневая структура ответа:

- `result` — присутствует при успешной операции, содержит сведения о платеже и чеке; `null` — в сценарии с ошибкой;
- `error` — `null` при успехе, в сценарии с ошибкой содержит числовой `error_id` и текст `text_error`.

Формально структура описана в артефакте [JSON-схема ответа — UC-10.2-1](schema-uc-10-2-payment.md).

## Пример успешного ответа

```json
{
  "result": {
    "paymentId": 12345,
    "amount": 150.0,
    "currency": "RUB",
    "status": "SUCCESS",
    "transactionId": "txn_abc123def456",
    "receiptNumber": "CH-789012",
    "paymentDate": "2024-06-15T14:30:00Z",
    "receiptDate": "2024-06-15T14:30:05Z"
  },
  "error": null
}
```

## Пример ответа с ошибкой

```json
{
  "result": null,
  "error": {
    "error_id": 400,
    "text_error": "Недостаточно средств на карте для оплаты."
  }
}
```

## Семантика полей

| Поле                   | Тип                              | Описание                                              |
| ---------------------- | -------------------------------- | ----------------------------------------------------- |
| `result.paymentId`     | integer                          | Внутренний идентификатор платежа в платформе парковки |
| `result.amount`        | number                           | Сумма операции в валюте `currency`                    |
| `result.currency`      | string                           | Код валюты в формате ISO 4217 (например, `RUB`)       |
| `result.status`        | string (enum)                    | Статус операции: `SUCCESS`, `FAILED`, `CANCELED`      |
| `result.transactionId` | string                           | Идентификатор транзакции у платежного провайдера      |
| `result.receiptNumber` | string                           | Номер фискального чека                                |
| `result.paymentDate`   | string (date-time, ISO 8601 UTC) | Момент подтверждения платежа провайдером              |
| `result.receiptDate`   | string (date-time, ISO 8601 UTC) | Момент фискализации чека                              |
| `error.error_id`       | integer                          | Числовой код ошибки                                   |
| `error.text_error`     | string                           | Человекочитаемый текст ошибки                         |
