## Назначение

Раздел подготовлен как единая точка входа для архитектуры данных и БД.

Он нужен, чтобы связать логическую модель предметной области, ERD, выбор СУБД, физическую модель, ограничения и эксплуатационные решения.

## Что хранить в разделе

Подходящие документы для этого каталога:

- ERD и доменные фрагменты связей;
- обоснование выбора PostgreSQL;
- принципы физической модели данных;
- решения по ключам, индексам и типам данных;
- решения по миграциям, аудиту, soft delete, журналированию и версиям схемы;
- архитектурные заметки по границе между ERD и физической моделью.
