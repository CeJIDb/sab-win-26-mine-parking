# Инструкция: субагент «Документ → Markdown» (doc-to-markdown)

Субагент **doc-to-markdown** создаёт markdown-файлы (.md) на основе загруженных документов: изображения, PDF, DOCX, TXT и другие форматы.

## Назначение

- Извлекать или описывать содержимое загруженных файлов.
- Формировать структурированный markdown: заголовки, списки, таблицы, цитаты — где это возможно.
- Для картинок: описывать текст и структуру и оформлять в виде .md (в т.ч. при отсутствии доступа к OCR — давать связное описание и предлагать разметку по смыслу).

## Когда использовать

- Нужно превратить протокол/отчёт из PDF или DOCX в .md для репозитория.
- Нужно оформить в markdown текст из TXT или скопированный из другого формата.
- Нужно по скриншоту/фото документа (схема, таблица, текст) получить .md-описание или расшифровку.
- Нужно единообразно привести несколько документов к формату .md.

## Как вызвать в Cursor

Указать субагента **doc-to-markdown** и приложить файл(ы), например:

- «Используй doc-to-markdown: конвертируй приложенный PDF в markdown и сохрани как docs/protocols/protocol-6.md».
- «doc-to-markdown: по этой картинке (схема/таблица) сделай markdown-описание».
- «Преобразуй приложенный DOCX в .md через субагент doc-to-markdown».

Файл субагента: `.cursor/agents/doc-to-markdown.md`.

## Входные данные

- Один или несколько файлов: изображения (PNG, JPG и др.), PDF, DOCX, TXT, другие документы.
- По желанию: имя/путь выходного .md, необходимость YAML-фронтматтера, язык текста.

## Ожидаемый результат

- Краткое пояснение: что обработано и какие ограничения (если есть).
- Готовый текст markdown-файла в блоке кода (удобно скопировать или сохранить).
- При указании пути — предложение сохранить результат по этому пути.

## Поддерживаемые форматы

| Формат       | Что делает субагент                                                                |
|--------------|------------------------------------------------------------------------------------- |
| TXT          | Сохраняет структуру, при необходимости вводит заголовки/списки.                   |
| DOCX         | Извлекает текст и разметку (заголовки, списки, таблицы) → markdown.                 |
| PDF          | Текст и по возможности структура → markdown; сложная вёрстка — с оговорками.       |
| Изображения  | Описание/«расшифровка» содержимого (текст, схемы, таблицы) в виде .md.             |
| Другие       | По возможности извлечь текст и оформить в markdown или описать ограничения.         |

## Соглашения

- Markdown — CommonMark/GFM.
- Для артефактов проекта при необходимости — YAML-фронтматтер в начале файла (`title`, `source`, `date` и т.д.).
- Язык по умолчанию — русский, если пользователь не указал иное.
