# Инструкция: субагент «Wireframe по артефактам» (wireframe-from-artifacts)

Субагент **wireframe-from-artifacts** предназначен для генерации **интерактивных wireframe-макетов** (HTML + CSS) по артефактам системного анализа и требованиям проекта. Макеты — без визуального дизайна (серые/чёрно-белые блоки), с семантичной разметкой и кликабельной навигацией между страницами.

## Назначение

- Читать артефакты из `docs/artifacts/` (карта навигации, контекстная диаграмма, User Story Map, концептуальная модель, глоссарий и др.) и требования из `docs/specs/`.
- Генерировать HTML + CSS файлы в стиле wireframe: семантика (`header`, `main`, `nav`, `section`, `form`), минималистичная стилизация (рамки, отступы), без бренда и цветового дизайна.
- Сохранять согласованность с картой навигации и сценариями из артефактов.
- Обеспечивать интерактивность: ссылки и кнопки ведут на соответствующие страницы, чтобы можно было проверить поток и структуру интерфейса.
- Оставлять в коде комментарии со связью к требованию/артефакту (например: «По Карте навигации: раздел 2.1»).

## Когда использовать

- Нужно получить интерактивный прототип интерфейса по уже готовой карте навигации и артефактам.
- Нужно проверить структуру экранов и навигационные потоки до этапа визуального дизайна.
- Нужно согласовать с заказчиком или командой набор страниц и блоков контента на основе User Story Map и контекстной диаграммы.
- Нужно подготовить wireframe для передачи дизайнеру или фронтенд-разработчику.

## Как вызвать в Cursor

Указать субагента **wireframe-from-artifacts** при создании задачи или в чате, например:

- «Используй субагент wireframe-from-artifacts: по карте навигации и артефактам из docs/artifacts/ создай wireframe для сайта-визитки и главной ЛК клиента».
- «wireframe-from-artifacts: сгенерируй интерактивный wireframe по всей карте навигации» (результат по умолчанию сохраняется в папку `ui/`).
- «wireframe-from-artifacts: добавь страницы раздела 3.3 ЛК админа (управление профилями клиентов и админов, аналитика) к существующему wireframe».

Файл субагента: `.cursor/agents/wireframe-from-artifacts.md`.

## Входные данные

- **Артефакты** (по умолчанию из `docs/artifacts/`): карта навигации, контекстная диаграмма, User Story Map, концептуальная модель, глоссарий; при необходимости — Opportunity Canvas, Impact Map и др.
- **Требования** из `docs/specs/` (если есть).
- **Уточнение задачи**: полный набор страниц по карте навигации или только указанные разделы (например, только сайт-визитка и ЛК клиента). По умолчанию результат сохраняется в папку **`ui/`** проекта.

## Ожидаемый результат

- Все файлы размещаются в папке **`ui/`** проекта.
- Набор **HTML-файлов** (один файл на логический экран/раздел) с семантичной разметкой и комментариями-ссылками на артефакты.
- Один общий **CSS-файл** в `ui/` (например `wireframe.css`) для единообразия: типографика, отступы, рамки, стили форм и кнопок в стиле wireframe.
- **Кликабельная навигация**: ссылки между страницами соответствуют карте навигации (относительные пути).
- Краткое описание использованных артефактов и соответствия разделов карты навигации созданным файлам.

## Соглашения по выходным артефактам

- Результат сохраняется в папку **`ui/`** в корне проекта.
- Имена HTML-файлов: латиница, kebab-case (например `index.html`, `client-dashboard.html`, `admin-tariffs.html` в `ui/`).
- В комментариях в коде указывать источник: «По Карте навигации: раздел N», «По User Story Map: …», «По глоссарию: …».
- Wireframe без визуального дизайна: серые/чёрно-белые блоки, без логотипов и цветового бренда; при необходимости плейсхолдеры для контента с пометкой «[по артефакту не определено]».
- JavaScript не требуется для базового прототипа — достаточно переходов по ссылкам.

## Связь с другими субагентами

- **systems-analyst** готовит и поддерживает артефакты (карта навигации, контекстная диаграмма, User Story Map и т.д.). Wireframe-субагент использует эти артефакты как вход.
- После согласования wireframe макеты могут передаваться дизайнеру для визуального оформления или фронтенд-разработчику для реализации на выбранном стеке.
