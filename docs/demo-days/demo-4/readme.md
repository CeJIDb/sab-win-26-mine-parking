## Назначение

Каталог нужен для сборки презентационных и вспомогательных материалов к Demo 4 без смешивания их с каноничной аналитикой проекта.

Сюда выносятся готовые слайды и сопровождающий текст выступления для показа.

Каталог следует общей структуре `demo-1/` ... `demo-5/` внутри `docs/demo-days/`.

## Текущий фокус demo-4

По текущему состоянию проекта в фокус Demo 4 логично включать:

- развитие архитектуры системы;
- DDD и C4;
- углубление ERD и решений по БД;
- материалы по информационной безопасности;
- готовность к постановке задачи для разработчика.

В текущей версии каталога уже подготовлен блок по `ES TO-BE с выделенными контекстами` как основанию для архитектурного разбиения системы.

## Содержимое каталога

- [Речь выступления по 4 слайдам ES TO-BE с контекстами](demo-4-es-tobe-contexts-speech-4-slides.md) — готовый текст выступления на 3–4 минуты для блока про `ES TO-BE с выделенными контекстами`.
- `slides/1.jpg` — титульный слайд блока про `ES TO-BE с контекстами`.
- `slides/2.jpg` — слайд с `Core`-контекстами и их основными связями.
- `slides/3.jpg` — слайд с `Supporting`- и `Generic`-контекстами.
- `slides/4.jpg` — заключительный слайд с итогами блока и переходом к следующему выступающему.

## Как использовать материалы

- Использовать [речь выступления по 4 слайдам ES TO-BE с контекстами](demo-4-es-tobe-contexts-speech-4-slides.md) как основу для устного рассказа.
- Показывать слайды из каталога `slides/` в порядке `1.jpg -> 2.jpg -> 3.jpg -> 4.jpg`.
- Каноничные аналитические и архитектурные артефакты при этом не дублировать в каталог Demo 4, а использовать по ссылкам из профильных разделов.

Если в Demo 4 будут добавлены другие блоки, например по ERD, C4 или ИБ, их лучше оформлять по той же схеме:

```text
demo-4/
  readme.md
  <topic>-speech-*.md
  slides/
    1.jpg
    2.jpg
    3.jpg
    4.jpg
```
