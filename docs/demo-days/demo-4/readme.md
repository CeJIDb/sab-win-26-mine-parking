## Назначение

Каталог нужен для сборки материалов к Demo 4 без смешивания презентационных файлов с каноничной аналитикой.

Сюда стоит класть сценарий выступления, выжимку ключевых изменений, адаптированные схемы и служебные материалы для показа.

Каталог следует общей структуре `demo-1/` ... `demo-5/` внутри `docs/demo-days/`.

## Предварительный фокус демо

По текущему состоянию проекта в фокус Demo 4 логично включать:

- развитие архитектуры системы;
- DDD и C4;
- углубление ERD и решений по БД;
- материалы по информационной безопасности;
- готовность к постановке задачи для разработчика.

## Что имеет смысл показать

- краткий narrative по пройденным этапам;
- текущий срез каноничных артефактов;
- что уже согласовано;
- что остается в работе до следующего рубежа;
- какие вопросы требуют обратной связи от заказчика или ментора.

## Рекомендуемая структура каталога

Пример наполнения:

```text
demo-4/
  readme.md
  demo-4-script.md
  demo-4-summary.md
  assets/
```

Каноничные артефакты при этом должны оставаться в профильных разделах репозитория, а сюда попадать как ссылки или адаптированные копии для показа.
