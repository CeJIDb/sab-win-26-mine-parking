# Слайды-перебивки между блоками — Демо 5

Спецификация слайдов-перебивок для финального доклада. Каждая перебивка — короткий слайд между смысловыми блоками, который служит «GPS-навигатором» для аудитории и местом передачи микрофона.

Связано:

- [Карта слайдов и спикеров](./slides-map.md) — общая структура доклада, спикеры, блоки.
- [История развития проекта](../../process/project-journey.md) — каноничный таймлайн пяти этапов проектирования, к которому привязаны перебивки.
- [Структура выступления](./talk-structure.md) — тайминг и регламент.
- [Передачи микрофона](./slides-map.md#передачи-микрофона) — фразы-мостики, использованные ниже.

## Назначение

Перебивки решают три задачи разом:

1. **Навигация.** Аудитория всегда видит, где мы в общей структуре проекта. Доклад идет 38 минут — без визуального якоря зритель теряется.
2. **Передача спикера.** На каждой перебивке имя следующего спикера и его цвет — зритель успевает переключить внимание до того, как новый человек начал говорить.
3. **Сюжетная связь.** Каждая перебивка озвучивает «вывод предыдущего блока → крючок следующего», поддерживая сквозную драматургию «4 проблемы → решения по блокам».

## Анатомия слайда-перебивки

Двухслойная композиция. Верхний слой постоянный, нижний меняется по этапу.

```text
┌─────────────────────────────────────────────────────────┐
│  [тонкая лента ВСЕХ 5 этапов сверху, ~15% высоты]       │
│  ✓ ✓ ● ○ ○        ← пройденные / текущий / будущие      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   [крупный фрагмент схемы текущего этапа, ~60% высоты]  │
│                                                         │
│   Этап N · Название этапа                  [имя · цвет] │
│   «Фраза-мостик в одну строку»                          │
│   Закрывает: P1 P2                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

Элементы:

- **Лента-прогресс сверху** — все 5 этапов одной полосой ([stages.jpg](../../process/assets/project-journey-stages.jpg) кроп по высоте). Состояния: пройденный (галочка + полупрозрачно), текущий (подсветка цветом спикера), будущий (серый контур).
- **Крупный план этапа** — соответствующий `stage-N.jpg` или его половина (для блоков, которые делят этап с соседом).
- **Заголовок и спикер** — имя этапа из project-journey, имя спикера справа в его цвете из [легенды slides-map](./slides-map.md#легенда-спикеров).
- **Фраза-мостик** — одна строка, из таблицы переходов.
- **Закрытые проблемы** — теги P1–P4 из [матрицы «проблема → решение по блокам»](./slides-map.md#матрица-проблема--решение-по-блокам). На перебивке показываем, какие проблемы уже закрыты к этому моменту (нарастающий итог).

Рекомендации:

1. **Соотношение сторон.** Все ленты узкие (~2.2:1 в случае коротких этапов, до 9.9:1 у общей ленты). На 16:9 слайде `stages.jpg` ляжет как полоса в верхние ~22% высоты — это и есть «лента-навигатор» из анатомии. Кропы этапов в крупный план на той же раскладке займут ~50–60% высоты слайда.
2. **Левая/правая половина.** Резать ровно пополам по ширине, **сверять по содержимому**: на stage-2.jpg граница 2.1↔2.2 не обязательно идет точно по центру. Перед сборкой в Google Slides — открыть оригинал и пометить пиксельную координату границы.
3. **Подсветка текущего этапа на ленте.** Самый простой способ — поверх `stages.jpg` рисовать прямоугольник цвета спикера с прозрачностью 30% над текущим участком. Координаты участков пропорциональны ширинам этапов: stage-1 ≈ 23%, stage-2 ≈ 36%, stage-3 ≈ 22%, stage-4 ≈ 16%, stage-5 ≈ 20% от общей ширины 21978 px.
4. **Галочки на пройденных этапах** — векторные иконки поверх ленты, не редактировать сам JPG.

## Открытые вопросы (требуют решения до сборки)

1. ~~**Раскладка спикеров блока 3.**~~ Решено: Артем держит этап 2 + этап 3.1 (CDM + UC как модели в SRS), Денис держит этап 3.2 (ФТ + НФТ + ограничения + ИБ). Слот «3.1 Модели для требований» добавлен в [slides-map.md](./slides-map.md) и [talk-structure.md](./talk-structure.md).
2. **Содержимое stage-5.** На схеме этапа 5 в project-journey могут быть нарисованы Kafka и RabbitMQ как варианты, тогда как Demo-5 идет по [no-brokers-pitch](./no-brokers-pitch.md) (REST + Outbox без брокеров, ADR-008). Решение: либо использовать как есть с устной оговоркой «рассматривали оба, выбрали Outbox» в T7, либо подготовить отредактированный кроп без Kafka/RabbitMQ.
3. **Параллельность этапов 2 и 3 в project-journey.** В исходной схеме project-journey этап 3 идет параллельно этапу 2. В demo-5 мы показываем чистый переход 2 → 3.1 внутри блока Артема, без оговорки про параллельность — на T3 голосом: «границы зафиксированы — переходим в этап 3, начинаем с моделей». Параллельность остается в project-journey.md как историческая правда, но в demo-5 не подсвечивается, чтобы не запутывать аудиторию.
4. **Длительность перебивок.** Тайминг в [talk-structure](./talk-structure.md#тайминг-доклада-на-сцене) сейчас рассчитан без отдельных слотов на перебивки. Реалистичная оценка — 10–15 секунд на слайд, итого ~2 минуты на 9 перебивок. Нужно либо вписать в существующий буфер 7 минут, либо ужать речь блоков.
